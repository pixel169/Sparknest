# SparkNest Capstone Project
## The Brief

SparkNest's operations team has been managing customer enquiries and support cases manually — spreadsheets, WhatsApp, and email. The CTO has approved a Salesforce implementation.

---

## What You Are Building

A working Salesforce org for SparkNest that covers:
- A clean data model for enquiries and support
- A secured org where each team member sees only what they should
- Automated workflows that remove manual steps
- A simple Apex layer for business logic that Flow cannot handle
- A working API endpoint that the SparkNest website can call
- Meaningful reports and dashboards for management
- One Agentforce agent that handles the most common support request

### Task 1 — Complete the Data Model

SparkNest's existing Enquiry__c object is missing a few things the business needs before go-live.

**Add these fields to Enquiry__c:**

| Field | Type | Requirement |
|---|---|---|
| Status | Picklist | New, In Progress, Closed — default New |
| Customer Budget | Currency | Min value ₹1,000 via validation rule |
| Closed Date | Date | Stamped automatically when Status = Closed |
| Priority | Picklist | High, Medium, Low — default Medium |

**Create a junction object — Enquiry Product Interest:**

A customer can enquire about multiple products in one conversation. Currently one Enquiry links to one Product — that is too restrictive.

Build `Enquiry_Product__c` as a junction object between Enquiry__c and Product2 with:
•  Lookup to Enquiry__c (required)
•  Lookup to Product2 (required)
•  Notes field (Text Area — 500 chars) — any specific question about that product
•  Quantity Interested (Number) — how many units the customer wants

### Task 5 — Flows

**Flow 1 — Screen Flow: Convert Enquiry to Case**

This is the Quick Action from Task 3. When a Sales Rep clicks Convert to Case on an Enquiry:

1. Show a screen displaying the Enquiry Subject and Contact Name
2. Ask the user to select Priority (High / Medium / Low) and enter a brief description
3. Create a Case — Subject = Enquiry Subject, Contact = linked Contact, Priority = selected value, Origin = Internal
4. Assign the Case to the Care Plan Support queue if Enquiry Type = Care Plan, otherwise assign to General Support queue
5. Update the Enquiry Status to Closed
6. Show a confirmation screen with the new Case number

**Flow 2 — Record-Triggered Flow: Auto-close Follow-up Task**

When an Enquiry's Status changes to Closed, find all open Tasks linked to that Enquiry and mark them as Completed. Run after the record is saved, only on update, only when Status changes to Closed.

**Flow 3 — Scheduled Flow: Weekly Enquiry Summary**

Runs every Monday at 8am. Queries all Enquiries created in the last 7 days. Creates one Task assigned to the Sales Manager with Subject: "Weekly Summary — [X] new enquiries this week." Use a text formula to build the subject dynamically.

### Task 6 — Apex

**6a — Trigger using Handler Pattern**

Build a trigger on Enquiry__c using the handler pattern:

- `EnquiryTrigger` — thin trigger, calls dispatcher
- `TriggerDispatcher` — routes to correct handler method
- `EnquiryTriggerHandler` — implements handler interface
- `EnquiryService` — contains all business logic

Logic to implement in the service class:

**Before Insert:**
- If Status is blank, default it to New
- If Customer Budget is less than ₹1,000, add a field error — "Customer budget must be at least ₹1,000"

**After Insert:**
- Create a follow-up Task due 3 days from today, assigned to the Enquiry owner, Subject = "Follow up: [Enquiry Subject]"

**Before Update:**
- When Status changes to Closed, stamp today's date into Closed Date

**6b — Test Class using Test Data Factory**

Build a Test Data Factory and a test class for EnquiryService with 75%+ coverage:

**6c — Async Apex: Batch to Close Stale Enquiries**

SparkNest wants all Enquiries that have been In Progress for more than 30 days with no activity to be automatically closed every night.
Schedule it to run nightly from Developer Console.
